# Issue Log (이슈 및 결함 관리대장)

* **사용 목적**: 보고된 시스템 버그와 비주얼 깨짐, 미확인 레이아웃 결함의 상태를 기록하고, 재현 증적을 병합하여 관리합니다.

---

## 1. 활성 결함 목록 (Active Issues)
*현재 해결을 위해 오픈되어 있는 버그들입니다.*

| Issue ID | Related File | Description | Expected vs Actual | Reproduction Evidence / Screenshots | Status |
| :--- | :--- | :--- | :--- | :--- | :--- |

## 2. 결함 기록 양식 및 수칙
* 버그 구현 전에 반드시 이슈 번호(예: `ISSUE-001`)를 기재하고, 예상 동작(Expected)과 실제 오류 동작(Actual)을 명확하게 분리하여 기재하십시오.
* 재현 스크립트나 콘솔 오류 로그 파일 경로를 'Reproduction Evidence' 열에 명시해야 패치 단계로 진입할 수 있습니다.
