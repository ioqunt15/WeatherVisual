# Lock Registry (파일 잠금 현황판)

* **사용 목적**: 다수 파일의 동시 다발적 충돌 수정을 막고, 현재 수정 중인 파일 경로의 독점권을 선언합니다.

---

## 1. 활성 파일 잠금 (Active Locks)
*현재 편집 모드로 들어간 파일 리스트입니다.*

| locked_file_path | Lock Owner (Role/Worker) | Acquired Timestamp | Reason |
| :--- | :--- | :--- | :--- |
| *없음* | - | - | - |

## 2. 잠금 취득 및 해제 규칙
* 구현자(Implementer)는 파일 쓰기 도구를 사용하기 전에 이 파일에 수정할 타겟 경로를 등록하고 락을 취득해야 합니다.
* 작업이 정상/비정상 종료되면 즉시 락을 해제(테이블에서 삭제)하고 다른 작업자에게 권한을 양도해야 합니다.
