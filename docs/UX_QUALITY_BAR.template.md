# UX Quality Bar Checklist (사용 편의성 자가 체크리스트)

* **사용 목적**: 사용자가 제품을 막힘없이 인지하고 사용하도록 하기 위한 UX 자가 진단서입니다.
* **작성자**: UX Architect / Interaction Designer
* **포함 사항**: 동선 가독성, 로딩/에러 상태 설계 충족 여부 체크리스트.

---

## 1. 내비게이션 및 흐름 (Flow & Control)
* [ ] 사용자가 임의의 복잡한 하위 작업 중에도 언제든 이전 화면으로 안전하게 돌아갈 수 있는 '뒤로가기' 수단이 존재하는가?
* [ ] 현재 내가 제품 내 어느 곳에 위치하고 있는지(Breadcrumbs, 활성 메뉴 강조 표시 등) 직관적으로 인지할 수 있는가?
* [ ] 중요 동작 직전에 실수 방지를 위한 적절한 컨펌/재확인 수단(모달, 더블 체크 등)이 구비되어 있는가?

## 2. 예외 및 대기 상태 피드백 (State Feedback)
* [ ] 네트워크 지연 발생 시 화면 전체가 흰색으로 멈추지 않고 스켈레톤 UI 또는 로딩 인디케이터가 명확히 보이는가?
* [ ] 작업이 성공 또는 실패했을 때 사용자에게 상황을 명확하게 텍스트(토스트 메시지, 배너 등)로 설명하는가?
* [ ] 빈 화면(Empty State)에 사용자가 다음 행동을 개시할 수 있도록 유도하는 CTA(Call-To-Action) 버튼이 포함되어 있는가?

## 3. 정보 가독성 (Readability)
* [ ] 문맥상 어색하거나 번역기 투의 문구가 필터링되었는가?
* [ ] 줄 간격(Line-Height)이 텍스트 크기 대비 적당하여 긴 문장을 읽을 때 피로감이 없는가?
