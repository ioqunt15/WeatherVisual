# Verification Policy (산출물 최종 검증 가이드라인)

* **사용 목적**: 기술적 동작 성공 및 시각적 레이아웃의 완성도를 다각도로 증명하여 작업 완료 신뢰성을 확보합니다.
* **작성자**: QA Verifier / Technical Evaluator
* **포함 사항**: 브라우저 캡처 가이드, 자동 검사 통과 규칙, 수동 검증 체크리스트 작성 규칙.

---

## 1. 다차원 검증 체계 (Multi-Layer Verification)
수행된 작업은 단순 '동작 확인'을 넘어 다음 세 영역에서 모두 검증되어야 합니다:
1. **시각적 완성도 검증**: Antigravity 브라우저 캡처 도구가 활성화되어 있을 시, 변경 전/후 스크린샷 이미지를 캡처하여 `system/verification_log.md`에 등재.
2. **반응형 호환성 검증**: 모바일(360px), 태블릿(768px), 데스크톱(1200px) 해상도 각각에서 수동 레이아웃 테스트 수행.
3. **자동화 테스트 검증**: 구성된 린터, 타입 체커 및 테스트 스크립트 무결성 패스.

## 2. 수동 검증 시 체크리스트 대체 가이드 (Fallback Manual Checklist)
테스트 환경이 불비하여 스크린샷 캡처나 자동 테스트 구동이 불가능할 경우, QA Verifier는 사용자가 직접 배포 환경에서 수동으로 확인할 수 있는 상세 **수동 검증 체크리스트**를 인도해야 합니다.
* **작성 서식**:
  ```markdown
  ### [검증 단계 명칭]
  * [ ] 수행 동작 1 (예: 로그인 버튼 클릭 시 에러 팝업 여부)
  * [ ] 관찰 대상 2 (예: 모바일 화면으로 줄였을 때 우측 상단 메뉴 햄버거 버튼 전환 확인)
  * [ ] 예상 결과 3
  ```

## 3. 검증 결과 기록 의무
* 모든 검증 내용은 반드시 `system/verification_log.md`에 작성되어야 하며, 통과 판정 근거 데이터가 없는 완료 보고는 에이전트의 규칙 위반으로 처리됩니다.
