# Responsive Rules (반응형 화면 설계 규칙)

* **사용 목적**: 모바일 해상도에서 화면 깨짐, 가로 스크롤 버그(Horizontal Scroll Overflow), 글자 뭉개짐 등을 예방합니다.
* **작성자**: Visual Design Director / UX Architect
* **포함 사항**: 기준 브레이크포인트, 모바일 가독성 텍스트 크기, 터치 영역 처리, 가로 넘침 방지 룰.
* **제외 사항**: CSS 미디어쿼리 하드코딩 리스트 자체.

---

## 1. 브레이크포인트 기준 (Breakpoints)
* **모바일 (Mobile)**: `320px` ~ `767px` (기본 레이아웃: 1열 배치)
* **태블릿 (Tablet)**: `768px` ~ `1023px`
* **데스크톱 (Desktop)**: `1024px` 이상

## 2. 모바일 가로 스크롤 방지 규칙 (No-Overflow Policy)
* **원칙**: 모바일 뷰포트에서 가로 스크롤 바가 생성되는 것은 즉각적인 **FAIL** 처리 사유입니다.
* **해결 방안**:
  - 테이블이나 가로형 카드 리스트는 모바일에서 vertical(세로형) 카드 스택으로 구조 변환을 하거나, 명시적으로 `overflow-x: auto`가 지정된 터치 스크롤 컨테이너 내부에 가둡니다.
  - 고정 너비(예: `width: 600px`) 속성 사용을 전면 금지하고, `max-width: 100%` 및 `box-sizing: border-box`를 기본 사용합니다.

## 3. 모바일 가독성 및 폰트 축소 (Typography Scaling)
* 데스크톱 대비 모바일에서의 타이틀 폰트 비율 축소를 정의합니다. (예: 데스크톱 H1이 48px이면 모바일에서는 32px로 조율)
* 모바일 본문 텍스트는 가독성을 확보하기 위해 최소 `14px` 이상을 유지해야 합니다.
