# Design Quality Bar Checklist (시각 디자인 품질 기준 자가 체크리스트)

* **사용 목적**: UI 최종 완성 선언 전에 개발자나 AI 에이전트가 자체적으로 디자인 수준을 자가 진단하는 성적표 역할을 수행합니다.
* **작성자**: Visual Design Director / Design Evaluator
* **포함 사항**: 비주얼 조화도, 시선 흐름, 템플릿 느낌 배제 항목 자가 평가 질문.

---

## 1. 시각적 계층 구조 및 레이아웃 (Layout & Rhythm)
* [ ] 화면을 처음 보았을 때 가장 중요한 핵심 정보와 버튼이 0.5초 이내에 시각적으로 구별되는가?
* [ ] 정보의 중요도에 따라 폰트 크기(H1, H2, Body)와 명도 대비가 균형 있게 적용되었는가?
* [ ] 컴포넌트 간의 가로/세로 간격이 규칙적이며 일률적인 그리드 시스템을 지키고 있는가?

## 2. 컬러 및 세부 디테일 (Colors & Details)
* [ ] 브랜드 정체성에 맞지 않는 촌스럽거나 과다한 원색, 난잡한 그라데이션이 적용되어 있는가? (있다면 감점 또는 FAIL)
* [ ] 다크/라이트 테마 변경 시 텍스트 명도 대비(Contrast Ratio)가 웹 접근성 가이드라인(최소 4.5:1)을 충족하는가?
* [ ] 테두리 둥글기(Radius) 값과 그림자 깊이가 화면 전반에 일관되게 나타나는가?

## 3. 맥락 맞춤성 및 템플릿 배제 (Specificity)
* [ ] 라이브러리의 기본 예제 화면을 그대로 복사 붙여넣기 한 듯한 "양산형 UI"는 아닌가?
* [ ] 해당 기능의 유즈케이스에 최적화된 화면 구성을 갖추었는가?
* [ ] 예외 케이스(자료 없음, 오류 상태)가 기본 빈 화면이 아니라 제품 톤앤매너에 어울리는 일러스트나 텍스트로 보완되었는가?
