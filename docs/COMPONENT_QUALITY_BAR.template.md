# Component Quality Bar (개별 컴포넌트 품질 가이드라인)

* **사용 목적**: 카드, 테이블, 입력 폼, 모달, 내비게이션 등 UI 단위 컴포넌트가 갖추어야 할 디테일 완성도 기준을 제시합니다.
* **작성자**: UX Architect / Interaction Designer
* **포함 사항**: 요소별 필수 구조, 내부 여백(Padding), 텍스트 오버플로우 처리 지침.
* **제외 사항**: 특정 라이브러리의 프롭스 명세서.

---

## 1. 카드 (Card)
* **내부 패딩**: 최소 `20px` 이상 확보하여 내용이 테두리에 달라붙지 않게 조절.
* **텍스트 처리**: 카드 제목이 길어지는 경우 한 줄 요약 후 말줄임표(...) 처리(`text-overflow: ellipsis`)를 하거나 자동 줄바꿈 설계.
* **그림자 및 테두리**: 의미 없이 모든 카드에 굵은 테두리와 강한 그림자를 동시 적용하지 말 것.

## 2. 테이블 (Table)
* **헤더와 바디 대비**: 테이블 헤더는 굵은 글씨 또는 옅은 배경색을 부여해 데이터 행과 명확히 구분되게 처리.
* **가로 폭 조절**: 열의 너비가 좁아 텍스트가 세로로 한 글자씩 쪼개져 렌더링되지 않도록 최소 너비(`min-width`) 또는 비율 조정.
* **정렬 규칙**: 텍스트는 좌측 정렬, 숫자는 우측 정렬, 상태 배지나 버튼은 중앙 정렬 표준화.

## 3. 입력 폼 (Form Input)
* **레이블 연동**: 모든 입력 칸은 반드시 텍스트 레이블(`label`)과 연결(`for` 및 `id` 매칭)되어 마우스 클릭 시 입력창으로 포커스가 이동해야 함.
* **플레이스홀더**: 플레이스홀더를 레이블 대신 사용하는 얕은 기법 금지. 플레이스홀더는 오직 입력 포맷 예시로만 활용.

## 4. 모달 및 내비게이션 (Modal & Navigation)
* **닫기 옵션**: 모달은 우측 상단 `X` 버튼, 배경 딤(Dim) 영역 클릭, `ESC` 키 입력의 세 가지 방식으로 닫힐 수 있어야 함.
* **포커스 락**: 모달이 열리면 포커스는 모달 내부에 갇혀야 하며 외부 요소를 건드릴 수 없어야 함.
