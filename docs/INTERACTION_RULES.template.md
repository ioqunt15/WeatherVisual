# Interaction Rules (인터랙션 동작 지침)

* **사용 목적**: 컴포넌트 동작, 인터랙션 피드백, 폼 유효성 검사 흐름 및 트랜지션을 규정하여 사용자 경험을 완성합니다.
* **작성자**: Interaction Designer
* **포함 사항**: 요소별 Hover/Focus/Active/Disabled 상태 비주얼 변화 규칙, 모바일 최소 터치 영역, 입력창 에러 메시지 노출 규칙.
* **제외 사항**: 특정 자바스크립트 프레임워크 애니메이션 라이브러리 설정 세부 코드.

---

## 1. 컴포넌트 상태별 비주얼 반응 (State Changes)
*모든 버튼, 링크, 입력창 등 대화형 요소가 가져야 하는 상태별 시각 피드백입니다.*
* **Hover**: (예: 배경색 10% 어둡게, 마우스 커서 pointer 변경)
* **Focus**: (예: 2px 보라색 outline 표시, 아웃라인 오프셋 2px)
* **Active**: (예: 98% 축소 트랜스폼 효과)
* **Disabled**: (예: 불투명도 40% 처리, `pointer-events: none` 선언)

## 2. 폼 요소 사용성 및 유효성 검사 (Form Usability)
* 입력창 내부 여백: 최소 좌우 `16px`, 상하 `12px` 보장.
* 에러 표시 시점: `onBlur` 포커스 아웃 시점에 인라인 에러 노출. (타이핑하는 도중 에러 남발 엄금)
* 유효성 성공 피드백: 초록색 테두리 및 체크 아이콘 표시 여부 정의.

## 3. 모바일 터치 대상 규칙 (Tap Target Guidelines)
* 모바일 환경의 모든 클릭 가능한 컴포넌트는 최소 `44px x 44px` 이상의 크기 또는 터치 가용 마진 영역을 확보해야 합니다.
* 인접한 버튼과의 최소 간격은 `8px` 이상을 강제합니다.
